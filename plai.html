Documentation

Feature guide

A product-facing guide to the Impossible Casino prototype, grouped by functional area so reviewers can see what exists and how each part behaves.

Site Shell

  • Purpose: Provides the customer-facing Impossible Casino wrapper with sticky navigation, branded actions and a consistent visual system across every route.
  • Customer navigation: Home, My Casino, Games, Live Casino, Bingo, Lotteries and Help Centre all route inside the single-page prototype without leaving the concept.
  • PlayAI tools bar: Site, AI Studio, AI UX, AI Experiment, AI Intelligence and Documentation sit above the customer site, keeping product-review tools separate from the live customer experience.
  • Tool separation: PlayAI pages use their own blue technical brand and hide the customer header and footer, while Site returns to the customer experience with the PlayAI bar still available.
  • Brand system: The elephant mark, star motif, deep purple header, pink primary actions and yellow highlights are reused across forms, lobbies, recommendations and lottery flows.
  • Page behaviour: Routes are handled in-page, so sections are shown and hidden by the prototype rather than loading separate documents.
  • Entry points: Join Now opens registration, Log In opens the login overlay, and Play actions use the same login path to keep the journey consistent.
  • Homepage offer experiment: The hero shows Casino, Bingo and Lottery welcome offers, each with a Join Now CTA that feeds offer performance into AI Experiment.
  • Header behaviour: The PlayAI bar remains fixed while scrolling, and customer navigation scrolls back to the top of the selected site area.
  • How to test: Move between the primary nav links and confirm the page changes in place while the header remains available at the top.

Game Lobbies

  • Purpose: Shows how casino, live casino and bingo content can be reorganised into controlled branded lobby pages using the same source game catalogue.
  • Grouped rails: Pages can show more than one carousel, grouped by featured picks, provider, content type or theme.
  • View all: Each rail can expand from carousel mode into grid mode, giving reviewers a way to compare compact browsing with fuller discovery.
  • Game cards: Cards show source labels, imagery, game name, provider and a clear Play action, using the same component style across all lobbies.
  • Play flow: Every Play button opens the branded login overlay, so the prototype demonstrates the handoff without navigating away.
  • Localised controls: Play, View all and Carousel labels follow the selected site language while analytics keeps the English event labels.
  • Fallback handling: If the live catalogue is unavailable, the prototype can still render a fallback set so the lobby remains reviewable.
  • Tracked signals: Route views, carousel usage, game selections and repeated clicks feed into AI Intelligence and AI UX recommendations.

My Casino

  • Purpose: Demonstrates a personal lobby builder where a customer or operator can compose a custom casino page from available games.
  • Edit mode: Users can add carousels, rename the lobby, search the catalogue, filter by source and provider, then add games into chosen rails.
  • Drag and drop: Games can be dragged from the library into carousels, reordered within a carousel, and carousels can be reordered on the page.
  • Library controls: Search, source and provider filters help narrow the game list before adding titles into the personal layout.
  • Carousel management: New carousels can be created, renamed through the lobby context and populated independently.
  • Saved lobby: Saving hides the builder tools and renders the result as a normal customer lobby with a single Edit lobby action.
  • Navigation sync: Renaming the lobby updates the My Casino label in the main navigation so the personal page name is reflected globally.
  • Localised builder: Save, Add carousel, Edit lobby, empty states, drag hints and provider filters all follow the selected site language.
  • Reviewer check: Build at least two rails, save the lobby, then re-enter edit mode to confirm the saved layout is still intact.

Registration

  • Purpose: Tests how the same registration field set performs when presented as a full form versus a guided multi-step flow.
  • Branded popup: Join Now opens an Impossible-styled registration overlay with the elephant panel, bubblegum colours and dummy completion.
  • A/B variants: The reviewer can switch between all-fields and step-by-step versions while the popup is open, without needing a page reload.
  • Field coverage: The flow includes account details, personal details, address, preferences, marketing consent, age confirmation and terms acceptance.
  • Step-by-step mode: Breaks the journey into account, personal details, address, preferences and confirmation stages to expose step drop-off.
  • All-fields mode: Shows the complete form at once for comparison against the guided journey and records the chosen variant.
  • Language coverage: Field labels, helper copy, step labels, button text, completion state and select options are translated with the main site language.
  • Progress tracking: Starts, step visits, field progress and completion are captured for funnel and variant comparison in AI Intelligence.
  • Tracking language: Registration analytics always records canonical English event names and labels, even when the visible form is translated.
  • Reviewer check: Open registration, switch variants while the modal is open, complete dummy fields and compare the variant metrics afterwards.

Login

  • Purpose: Provides a branded sign-in handoff for Play actions and header login, modelled as an overlay rather than a separate page.
  • Overlay flow: Login uses a split-panel presentation inspired by the reference flow, adapted to the Impossible brand with dummy submission.
  • Game handoff: Game play actions launch login so the customer stays inside the concept while the prototype still captures intent to play.
  • Form behaviour: Email, password and continue interactions are tracked separately so drop-off can be reviewed at field level.
  • Language coverage: Heading, helper copy, email and password labels, Continue, Join, Close and success messages follow the selected customer-site language.
  • Login funnel: Start, email field, password field and continue button events are visualised in AI Intelligence with first-step and previous-step percentages.
  • Tracking language: Login analytics remains in English, so the funnel stays comparable across English and Spanish sessions.
  • Dummy completion: Submission is intentionally local only; the goal is to test flow clarity, not account authentication.
  • Reviewer check: Trigger login from both the header and a game card to confirm both entry points feed the same overlay and analytics trail.

Lotteries

  • Purpose: Adds lottery functionality in the same bubblegum brand, using a richer ticket-building experience rather than the source layout.
  • Ticket builder: Customers select 5 numbers and 2 stars per line across multiple ticket cards, with clear selected states and delete controls.
  • Fast selection: Dice buttons randomise an individual line, while quick picks can fill one, three or all cards in a single action.
  • Participation controls: Customers choose draw days, number of weeks and whether to auto-renew before proceeding.
  • Validation cues: Incomplete lines keep the checkout state guarded, making it clear why a user cannot continue yet.
  • Checkout summary: The panel calculates selected lines and shows a Go to checkout action; Add to cart is intentionally excluded for this concept.
  • Language coverage: Ticket status, Stars, draw days, weeks, auto-renew, checkout warnings and button states all respond to the selected site language.
  • Tracked signals: Draw day, duration, quick picks and dice usage feed the lottery charts in AI Intelligence.
  • Tracking language: Lottery analytics keeps English labels for days, durations, quick picks and dice usage while the customer UI can be translated.
  • Reviewer check: Fill one line manually, use dice on another, quick-pick all remaining cards, then change duration and draw-day settings.

Language and Localisation

  • Purpose: Demonstrates how the customer-facing site can switch language without changing PlayAI tools or analytics semantics.
  • Footer selector: A compact language control sits in the footer action row, currently supporting English and Spanish.
  • Default language: English remains the default and is restored when the selected language is unavailable.
  • Site-only scope: Primary navigation, homepage, lobbies, My Casino, lotteries, account flows, Help Centre and footer copy are translated.
  • PlayAI exclusion: AI Studio, AI UX, AI Experiment, AI Intelligence, Documentation, PlayAI navigation and implementation briefs remain in English.
  • Brand exclusions: Impossible Casino, game names, provider names, GamCare, GAMSTOP and support phone numbers are preserved.
  • English tracking: Events, funnel labels, click labels, route labels and analytics payloads are recorded in English regardless of the displayed language.
  • Performance cache: The language layer caches merged dictionaries, repeated placeholder replacements and translatable DOM nodes to make switching faster.
  • Future languages: The language normaliser and dictionary cache are ready for additional language dictionaries such as French.
  • Reviewer check: Switch to Spanish, confirm visible customer-site copy changes, then check AI Intelligence to confirm tracking labels remain English.

Help Centre

  • Purpose: Provides a friendly customer-facing support area for games, My Casino, lotteries, account flows and safer play.
  • Main navigation: Help Centre appears at the end of the customer navigation, while Promotions has been removed from the main navigation.
  • FAQ structure: Topics are organised as expandable questions and answers with polished purple controls that match the Impossible brand.
  • Customer-only scope: The Help Centre avoids PlayAI and product-tool explanations so it reads like a real customer support area.
  • Safer play: Answers explain 18+ messaging, limits, breaks, GamCare, the National Gambling Helpline and GAMSTOP signposting.
  • Language support: FAQ headings, questions and answers participate in the same customer-site language switch as the rest of the site.
  • Reviewer check: Open Help Centre, expand several questions, switch language in the footer and confirm the customer-facing FAQ updates.

Studio

  • Purpose: Lets reviewers type natural-language change requests and turn them into Codex-ready implementation briefs.
  • Prompt composer: Prompts can be scoped to the whole site, homepage, My Casino, lotteries, forms or workspace tools.
  • Context capture: Reviewers can capture the current route as context and submit a prompt without leaving the customer page.
  • Submit flow: Queue for Codex saves a Codex-ready implementation request in Prompt history, while Preview brief only creates the brief without queuing it.
  • Prompt queue: Requests show draft, queued and executed states with source, scope, captured context and reopen controls for continued iteration.
  • Change preview: The studio generates implementation guidance, brand constraints and acceptance checks before any change is queued.
  • Recommendation handoff: UX recommendations can be sent directly to Studio as a scoped Codex-ready prompt.
  • Prompt demo path: Reviewers can use AI Studio as a scripted demo entry point for changes such as adding new site languages.
  • Reviewer check: Enter a prompt, preview the brief, queue it for Codex, then confirm it appears in Prompt history.

Widgets and Feature Switches

  • Purpose: Provides a PlayAI-controlled library of reusable customer-site components that can be switched on for demos without changing core page content.
  • Service banner: A configurable notice can appear above the main customer navigation for operational messages such as payment disruption or planned maintenance.
  • Widget controls: Reviewers can toggle display, choose placement, edit message text and set tone to Notice, Warning or Resolved.
  • Feature switches: Optional customer journeys can be controlled by market rule, making it possible to demonstrate UK-specific compliance steps separately from other markets.
  • Deposit limits: The first switch can show a post-registration deposit-limit component with daily, weekly or monthly defaults and UK/optional/off market behaviour.
  • Tracking: Widget display, dismissal, switch saves and deposit-limit choices are recorded as customer-site analytics events.
  • Roadmap: Future widgets can include offer rails, trust modules and experiment slots using the same configuration pattern.
  • Reviewer check: Enable the service banner, return to Site, confirm it appears above the customer header, then enable deposit limits and complete registration.

AI UX

  • Purpose: Turns behavioural signals into reviewable UX recommendations, helping reviewers spot friction without manually reading the event log.
  • Recommendations: Repeated clicks, unclear game handoffs, no-result searches and unfinished journeys generate grouped suggestions with evidence.
  • Grouping: Similar suggestions are combined so one recommendation can represent several related signals and components.
  • Review workflow: Suggestions can be approved, rejected or applied as safe prototype changes, giving the reviewer a simple decision path.
  • Codex handoff: Send to AI Studio converts a recommendation into a queued implementation prompt with evidence and affected component context.
  • Element previews: Where possible, the related component appears as a visual preview on the right of the recommendation card.
  • Badge colours: High uses red, medium amber and low blue; rejected is red, pending amber and accepted/applied green.
  • Badge meaning: High, medium and low priorities use distinct colours, while pending, rejected and accepted statuses show decision progress.
  • Reviewer check: Create repeated clicks or no-result searches, generate suggestions, then approve or reject grouped recommendations.

AI Experiment

  • Purpose: Documents active customer-site A/B tests so reviewers can see what is being tested, whether it has enough data and what the current signal suggests.
  • Experiment cards: Each test gets its own card with status, sample size, conversions, conversion rate, average time-to-convert, significance and outcome.
  • Dependency map: Tests that may influence one another are called out before the cards so reviewers understand when results should be interpreted carefully.
  • Report export: The experiment view can copy a concise report covering status, sample size, conversion, significance, warnings and variant performance.
  • Registration test: Compares the all-fields registration popup against the step-by-step registration popup using starts, completions and completion time.
  • Login handoff test: Monitors whether header login or game-card login entry points create clearer progress through the branded login overlay.
  • Lottery aids test: Tracks manual ticket completion against dice and quick-pick assisted selections to understand which tools help users reach checkout.
  • Homepage offer test: Compares Casino, Bingo and Lottery offer impressions, Join Now clicks and registration completions from the homepage hero.
  • Data scope: Results use customer-site analytics only; PlayAI route views and tool interactions are excluded from experiment calculations.
  • Significance: Cards show whether a result is collecting data, directional or strong enough to act on based on sample size and conversion gap.
  • Reviewer check: Run a few registration, login and lottery journeys, then open AI Experiment to confirm the cards update without changing the customer site.

AI Intelligence

  • Purpose: Provides the evidence layer for the prototype, showing what users did and where they dropped out.
  • Persistent metrics: Events and metrics remain available until cleared by the reviewer, so longer test sessions can build a useful picture.
  • Site-only data: PlayAI routes and tool interactions are filtered out of the customer-site intelligence view so analysis focuses on the website experience.
  • Segment filters: Reviewers can filter by customer route and event family to inspect navigation, games, registration, login, lotteries or widgets independently.
  • Metric definitions: The dashboard explains the top-level metrics so reviewers know what is counted and why.
  • English semantics: Analytics keeps event names, route names, click labels, component names and funnel labels in English, even when the customer UI is translated.
  • Charts: Navigation, routes, selected games, carousel usage, lottery choices and form funnels are visualised with bars, pies and funnel charts.
  • Lottery analysis: Duration, draw day, quick pick and dice usage are shown as charted behaviour signals.
  • Carousel analysis: Most used carousels are charted alongside selected games and views by route.
  • Funnel detail: Registration and login funnels show both percentage from first step and percentage from previous step to reveal absolute and step-by-step drop-off.
  • Conversion comparison: Registration variants show conversion rate and time-to-convert side by side for A/B review.
  • Reviewer actions: Suggestions can be refreshed from this area and all captured events can be cleared when starting a fresh test run.
  • Event timeline: Recent events remain visible so reviewers can confirm interactions are being captured in the expected order.
  • Reviewer check: Run through a short test journey, visit AI Intelligence, confirm charts update, then clear events before a fresh scenario.

Games and lobbies

How do I find a game?

Use the main navigation to browse Games, Live Casino or Bingo. Each lobby is grouped into themed rails, and View all expands a carousel into a larger grid when you want to browse more at once.

What happens when I press Play?

Play opens the Impossible login overlay. This keeps the demo inside the branded experience and shows where a real customer would sign in before continuing.

Why do some games appear in more than one place?

Games can appear in featured rails, provider groups or personal carousels. That helps demonstrate how the same catalogue can be reorganised for different journeys.

My Casino

What is My Casino?

My Casino is a personal lobby builder. You can create carousels, add games, reorder games and save the result as a normal-looking lobby.

How do I add games to a carousel?

Open Edit lobby, search or filter the game library, then drag a game into the carousel you want. You can also reorder games once they are inside a carousel.

Can I rename My Casino?

Yes. In edit mode you can change the lobby name. The main navigation updates too, so the personal page name is reflected across the site.

Lotteries

How do I complete a lottery ticket?

Select 5 numbers and 2 stars on a ticket card. A line needs both parts before checkout becomes available.

What do the dice and quick picks do?

The dice randomises one ticket line. Quick picks can fill one, three or all cards, which is handy if you want a faster route to checkout.

Can I choose draw days and weeks?

Yes. Choose Tuesday, Friday or both, then select the number of weeks. The summary updates with complete lines, renew status and total cost.

Account, login and registration

Is this a real account sign-up?

No. Registration and login are concept flows only. They are designed to show the customer journey, and no account data is submitted.

Why are there two registration layouts?

The prototype compares a single-page form with a step-by-step flow. AI Intelligence can show conversion and time-to-complete differences between the two variants.

Will my browser show saved emails or passwords?

The forms are set up for demo use and avoid normal saved credential prompts where possible, so personal details are less likely to appear during a presentation.

Safer play

What safer gambling information is included?

The footer includes 18+ messaging, reminders to set limits and take breaks, plus support signposting for GamCare, the National Gambling Helpline and GAMSTOP.

Is Impossible Casino a real gambling site?

No. It is a branded concept prototype. The safer play content is included to demonstrate the kind of responsible information a UK-facing experience should make easy to find.

Where can someone get support?

Anyone affected by gambling can contact the National Gambling Helpline on 0808 8020 133 or visit GamCare. GAMSTOP can help with online self-exclusion.
